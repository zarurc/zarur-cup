## Supabase

**Why:** Postgres database + Auth (anonymous sessions for invite-code login) + Storage. Every Phase 1 plan after this one talks to Supabase.

### Dashboard configuration

| Step | Where | What |
|------|-------|------|
| 1 | https://supabase.com/dashboard -> **New Project** | Name: `zarur-cup`. Region: **US-East-1** (closest to Vercel's default edge). DB password: store in 1Password. |
| 2 | wait ~2 min | Provisioning |
| 3 | **Project Settings -> API** | Confirm you see BOTH key formats: legacy (`anon` / `service_role`) AND new (`sb_publishable_*` / `sb_secret_*`). **Use the new format only.** |
| 4 | **Account -> Access Tokens** | Create a token named `zarur-cup CLI`. Copy once. Cannot be re-read. |
| 5 | **Project Settings -> General** | Note the **Reference ID** (e.g. `abcdefghijklmnop`) |

### Environment variables (server-side and client-side)

These go into `.env.local` at the project root. **`.env.local` is gitignored — never commit it.**

| Name | Source | Scope | Notes |
|------|--------|-------|-------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase Dashboard -> Project Settings -> API -> Project URL | client + server | shape `https://<ref>.supabase.co` |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | Supabase Dashboard -> Project Settings -> API keys -> `sb_publishable_*` | client + server | NEW key format only |
| `SUPABASE_SECRET_KEY` | Supabase Dashboard -> Project Settings -> API keys -> `sb_secret_*` | **server only** | NO `NEXT_PUBLIC_` prefix. Service-role authority. Consumed only by `src/lib/supabase/service.ts` (which is `import 'server-only'`). |
| `SUPABASE_ACCESS_TOKEN` | Supabase Dashboard -> Account -> Access Tokens | local CLI / CI | Needed for non-TTY `supabase db push` and `supabase link`. |
| `SUPABASE_PROJECT_REF` | Supabase Dashboard -> Project Settings -> General -> Reference ID | local CLI / CI | Used by `npm run db:link`. |
| `INVITE_CODE` | choose any string | **server only** | Family invite code. Plan 01-04 reads this. Placeholder is fine for now. |
| `ADMIN_DISPLAY_NAME` | choose any string | **server only** | Admin bootstrap (D-04). First user whose display name matches this gets `is_admin=true`. Placeholder is fine for now. |


### Recovery procedure if `.env.local` is lost

1. Reissue `SUPABASE_ACCESS_TOKEN` (Account -> Access Tokens -> Generate new -> revoke old).
2. The other four Supabase values are read-only on the dashboard — re-fetch them from Project Settings -> API.
3. Pick any value for `INVITE_CODE` and `ADMIN_DISPLAY_NAME`. They are not stored upstream.

### Cost ceiling

Free tier covers the project comfortably:
- 500MB DB storage (we'll use ~2-5MB)
- 50K MAU (we'll have ~15)
- 200 concurrent realtime (we'll have ~15 at peak)
- 5GB egress / month (we'll use <100MB)

The one risk on free tier is **auto-pause after 7 days of no API requests** — Plan 01-05 mitigates with Vercel Cron heartbeat every 3 days.
