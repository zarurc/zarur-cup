    **Step E: Create `tests/e2e/fixtures/api.ts`** — POSTs the savePrediction Server Action against the running app to assert RLS rejection.

    Server Actions in Next.js are addressable via POST to the page URL with an `action` header; the canonical wire format is documented at https://nextjs.org/docs/app/building-your-application/data-fetching/server-actions. The simplest approach for a test is: use Playwright's `request.fetch` API to POST against the action endpoint with cookies from the user's authenticated context, and observe the response.

    ```typescript
    // tests/e2e/fixtures/api.ts
    import type { APIRequestContext, BrowserContext } from '@playwright/test';

    export type PredictionWriteResult =
      | { ok: true }
      | { ok: false; error: 'locked' | 'validation' | 'network' | 'unauthenticated' | 'rls_denied' };

    /**
     * Attempts a post-lock prediction write via the savePrediction Server Action.
     * Uses the user's existing browser cookies (so the request is authenticated
     * as the test user, not anon).
     *
     * The smoke MUST observe one of:
     *   - HTTP 4xx (RLS rejection bubbles out as a 403 or 400)
     *   - JSON body { ok: false, error: 'locked' | 'rls_denied' | 'validation' }
     *
     * If we see HTTP 200 + { ok: true }, the lock was bypassed — D-30 + ROADMAP
     * §Success Criterion 5 are violated, and the smoke FAILS.
     *
     * Implementation note: Server Actions require a request body shaped per the
     * Next.js wire protocol (multipart with action ID). For simplicity, the
     * helper instead hits an explicit JSON API route — `app/api/test-only/save-prediction/route.ts`
     * — that the planner adds in this plan IF and only if Server Actions can't
     * be invoked directly from request.fetch. If the Server Action call works
     * directly, that's preferred (no new public route).
     *
     * For Phase 2 v1, the SIMPLEST reliable path is: have the smoke navigate
     * to /en/matches (the post-lock fixture row is locked in the DOM — no
     * stepper testid present) and ALSO call a thin `/api/_test/save-prediction`
     * route that internally calls savePrediction(). This route is dev-only,
     * gated by NODE_ENV === 'test', and returns the raw action result.
     */
    export async function attemptPredictionAgainstLockedFixture(
      request: APIRequestContext,
      ctx: BrowserContext,
      fixtureId: string,
      home: number,
      away: number,
    ): Promise<PredictionWriteResult> {
      // Use the player's existing cookies from ctx (same origin, same user).
      const response = await request.post('/api/_test/save-prediction', {
        data: { fixture_id: fixtureId, home_score: home, away_score: away },
        // playwright will use the context's cookies by default
      });

      if (!response.ok()) {
        // HTTP 4xx — RLS rejected. Treat as { ok: false, error: 'rls_denied' }.
        return { ok: false, error: response.status() === 403 ? 'rls_denied' : 'network' };
      }
      const body = await response.json();
      return body as PredictionWriteResult;
    }
    ```
