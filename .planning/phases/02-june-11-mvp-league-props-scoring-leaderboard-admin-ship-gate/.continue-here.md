## Critical Anti-Patterns

| Severity | Pattern | Why |
|----------|---------|-----|
| **BLOCKING** | Next.js App Router: folder names prefixed with `_` are PRIVATE folders and excluded from routing entirely. Even files inside (`route.ts`, `page.tsx`) are silently dropped from the build. | Production-safety boundary for test routes goes in a `NODE_ENV` + secret env-var check INSIDE the route, NOT in the folder name. Use `test/` not `_test/`. See https://nextjs.org/docs/app/getting-started/project-structure#private-folders. |
| **non-blocking** | npm install on Mac when CI is Linux can produce platform-incomplete lockfiles | Use `npm install --include=optional` (or regenerate in a Linux Docker container) to capture all platform-conditional deps |
| **non-blocking** | `supabase-js` realtime client requires native WebSocket; Node 20 lacks it | Pass `realtime.transport = ws` to `createClient()` in tests until CI moves to Node 22+ |
| **non-blocking** | GitHub Actions runners are IPv4-only → cannot reach Supabase direct connection (IPv6) | Always use the Supabase Session pooler URL in CI (`*.pooler.supabase.com:5432`) |
| **non-blocking** | Playwright's top-level `request` fixture is an isolated `APIRequestContext` with NO cookies. Use `browserContext.request` to share cookies with a specific context. | The misleading comment "Playwright passes cookies automatically" cost us iteration #5. Helpers should take a `BrowserContext` so the footgun can't reappear. |
