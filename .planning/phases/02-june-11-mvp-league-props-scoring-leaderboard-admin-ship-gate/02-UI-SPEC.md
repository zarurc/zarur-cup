# Phase 2 — UI Design Contract

> Visual and interaction contract for the June 11 MVP: matchday feed with inline ± steppers, locked-state capsule, post-result side-by-side reveal, props page, leaderboard with inline expand, and the admin scoring/roster/props/tournament-tree surface. Extends Phase 1 design tokens (`var(--zc-*)`, Heebo/Inter, 44px touch targets, Tailwind v4 logical properties, bottom tab bar). Generated by gsd-ui-researcher 2026-05-24; verified by gsd-ui-checker before planning.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (hand-built on Tailwind v4.3 + project conventions) |
| Preset | not applicable |
| Component library | none — bespoke primitives in `src/components/ui/*`, layout chrome in `src/components/layout/*` |

## Spacing Scale

Declared values (multiples of 4 — matches Tailwind default `--spacing: 0.25rem` scale):

| Token | Value | Tailwind | Usage in Phase 2 |
|-------|-------|----------|------------------|
| xs | 4px | `1` | Gaps between flag and team name; `+pts` badge inner padding; gap between two score digits inside a capsule. |
| sm | 8px | `2` | Compact inline gaps — between countdown clock segments, between locked capsule and 🔒 emoji, between leaderboard rank and name, between bottom tab bar and safe-area-inset. |
| md | 16px | `4` | Default — match-row inner padding (`pi-4`), date-header padding, prop card inner padding, leaderboard row vertical padding (`pbs-4 pbe-4`), section gutters on `/matches`, `/props`, `/leaderboard`. |
| lg | 24px | `6` | Padding between admin tabs and integrity widget; gap between date groups; flag grid outer padding on `/props`. |
| xl | 32px | `8` | Padding between major page sections; vertical gap above the first match group; bottom padding above the tab bar so content never tucks under it. |
| 2xl | 48px | `12` | Top spacing under sticky chrome on `/matches` (header 56 + countdown 40 = ~96 sticky offset; page content begins at `mbs-12`); empty-state card vertical padding. |
| 3xl | 64px | `16` | Reserved — bottom-of-feed sentinel; not commonly used in Phase 2. |

**Exceptions:**
- **Touch target floor:** Every tappable element (stepper buttons, flag-grid cells, leaderboard row hit area, locale pill, tab cell) must compute to at least 44×44px even when its visual is smaller (use padding to hit the floor).
- **Countdown banner height:** 40px (`bs-10`) — between `sm (8)` and `md (16)` doubles. This deliberately breaks the strict 4-multiple scale because the banner sits between the 56px header and the 16px content top-padding, and 40px keeps the total sticky chrome at a round 96px. Documented as the only exception.
- **Locked capsule height:** 32px (`bs-8`) — multiple of 4, but worth flagging explicitly because the surrounding match row is 64–80px tall; the capsule is a child element, not a row anchor.

## Typography

Three roles × two weights = the entire Phase 2 type system. Weights MUST be 400 (normal) and 700 (bold) only — these are the two weights Phase 1 self-hosts for both Heebo and Inter. No 500/600/800.

| Role | Size | Weight | Line Height | Tailwind | Usage |
|------|------|--------|-------------|----------|-------|
| Body | 16px (1rem) | 400 normal | 1.5 | `text-base` | Team names, prop prompts, leaderboard names, date headers (the textual portion), admin metric labels, empty-state body. |
| Label | 14px (0.875rem) | 400 normal (700 when emphasizing) | 1.45 | `text-sm` | `Saved/נשמר` indicator, kind labels in `+pts {kind}` badges, kickoff time under each match row, "League / Bracket / Props" subtotal labels in the leaderboard expand, tab labels (already established Phase 1), `+pts {kind_label}` per-player badge text. |
| Display | 24px (1.5rem) | 700 bold | 1.2 | `text-2xl` | Score digits inside the stepper (the live "0" / "1" / "2"); score digits inside locked capsule (`2:1 🔒`); leaderboard total points (right-aligned, tabular-nums); countdown timer digits (`02:14:33`). All MUST set `font-variant-numeric: tabular-nums` so they don't reflow on tick. |

**Headings deliberately omitted.** Phase 1 uses `text-xl` (20px) bold for empty-state and `/me` headings via the `EmptyStateCard` component. Phase 2 reuses that component as-is. No new heading sizes introduced. The page-level h1 on `/matches`, `/props`, `/leaderboard` is visually hidden (`sr-only` for a11y); the sticky chrome provides the visual identity.

**Numeral direction (bidi):**
- Score capsules, countdown timer, leaderboard total, `+pts N` badges, `Total Predictions: N` metric must render LTR even on `/he/*` pages. Wrap them in `<span dir="ltr">` (or `<bdi>` if the surrounding text is also bidi-fragile). This satisfies I18N-06 and the Phase 1 D-04 explicit-`dir="ltr"` on the stepper number input.
- Hebrew date string `שבת · 14 ביוני` is naturally RTL — let `<html dir="rtl">` handle it.

## Color

Phase 1's existing 60/30/10 split (carried forward verbatim):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fafaf7` (`--zc-surface`) | Page background. The whole-app cream surface set on `<body>`. |
| Secondary (30%) | `#ffffff` (`--zc-card`) | Cards, sticky header, sticky countdown banner, tab bar, match rows, prop cards, leaderboard rows, empty-state card, all "elevated" surfaces. |
| Accent (10%) | `#f59e0b` (`--zc-accent`) | RESERVED-FOR list below. Never used as a default button color. |
| Destructive | `#dc2626` (`--zc-destructive`) | Inline error text under stepper on save-failure; admin "Database Sync ✗" failure state; only NEGATIVE-event coloring. |

**Accent (`--zc-accent` `#f59e0b`) reserved for — explicit list, exhaustive:**
1. Active-tab indicator bar (4×8px above active label) — already Phase 1.
2. Form-field focus ring (`--zc-ring` is the same hex by design) — already Phase 1.
3. Focus rings on stepper +/− buttons, flag-grid cells, leaderboard rows, admin toggle, "Save Result" button.
4. The currently-selected flag in the props team-picker (ring + opacity override on selected cell).
5. The "Score Entry Mode" pill in the admin View/Entry toggle when it is the active mode.
6. The transient saved-checkmark glyph color (`✓` rendered in `text-[var(--zc-accent)]`).
7. The countdown timer's tens-of-seconds digit when ≤ 60 seconds remain to nearest kickoff (escalation cue — not blinking).

Accent is **never** used for: default text, default borders, body buttons, lock icons, the rank pill on the leaderboard, or the "+pts" badge color. Those use `--zc-primary` or `--zc-muted-foreground`.

**Phase-2-extended semantic tokens (NEW — added to `:root` in `globals.css` during planning):**

| Token | Value | Usage |
|-------|-------|-------|
| `--zc-score-exact` | `#0a2540` (alias of `--zc-primary`) | The `+4 exact` badge background (or text color on a transparent badge). Carries the primary-color weight to read as "the best outcome." |
| `--zc-score-good` | `#525252` (alias of `--zc-muted-foreground`) | The `+3 goal-diff` and `+2 winner` badge text. Reads "neutral correct." |
| `--zc-score-miss` | `#a3a3a3` (new, gray-400-ish) | The `+0 miss` and `+0 (no pick)` badge text. Reads "muted, didn't earn." |
| `--zc-lock` | `#525252` (alias of `--zc-muted-foreground`) | The locked-capsule background, the dashes `- : -`, the 🔒 emoji opacity. NOT destructive — locked is a neutral state. |
| `--zc-integrity-ok` | `#16a34a` (new, green-600) | Admin integrity widget `Database Sync: OK ✓` and the green check glyph. The single green in the system — appears ONLY in the admin integrity widget. |

**Rationale for not adding score-event colors as net-new accent palette:** The four score-kinds (exact / goal-diff / winner / miss) communicate magnitude through TEXT (`+4 exact`, `+3 goal-diff`), not color. Adding four new color tokens would (a) blow the accent budget, (b) fail color-blind users, (c) not survive Hebrew translation of the kind label. The badge background stays neutral; the magnitude lives in the number and the word.

**Rationale for green only in the admin widget:** The integrity widget is the only surface where a player would never see it (admin-only per D-15), so introducing a green here doesn't pollute the player palette. If the family ever sees a green pixel in this product, the admin is looking at a working database.

## Component Anatomy (Phase 2 net-new primitives)

These are NOT in the template but are MANDATORY for the executor — the planner must reference them in tasks. Specs are prescriptive, not exploratory.

### 1. Stepper (`MatchRowStepper.client.tsx`)
- **Anatomy:** `[− 44×44] [score 24px display, tabular-nums] [+ 44×44]`. Three-element flex row, gap `xs (4px)`.
- **Buttons:** 44×44px squares, rounded `rounded-xl`, border `border-[var(--zc-border)]`, background `bg-[var(--zc-card)]`, label is the literal Unicode character `−` (U+2212 MINUS SIGN) and `+` (U+002B PLUS SIGN) at 20px / `text-xl` / `font-bold` / color `text-[var(--zc-primary)]`. Centered, line-height 1.
- **Tap behavior:** Tap = ±1. Long-press = repeat at 150ms intervals after a 400ms initial delay (cap at score=9 / floor=0). Disabled visual = `opacity-40` + `pointer-events-none` (used at the bounds). Active depress = `bg-[var(--zc-muted)]` for 100ms.
- **Score display between buttons:** `<span dir="ltr" class="bs-11 min-is-8 text-2xl font-bold text-[var(--zc-primary)] tabular-nums text-center">{score}</span>`. Width is min 32px (`min-is-8`) — accommodates 1 or 2 digits without reflow.
- **Number input (a11y):** Use a hidden `<input type="number" inputMode="numeric" min="0" max="9" dir="ltr" class="sr-only">` driven by the stepper state for screen-reader users and form submission semantics.
- **States:** idle / saving (no visual change — keep optimistic) / saved (parent shows the inline checkmark, stepper unchanged) / save-failed (parent reverts value + shows inline error; stepper does not get its own error border).
- **RTL:** Flex direction stays LTR-DOM; `<html dir>` flips visual order so the home-team `−` is always closest to the home name. No `flex-row-reverse`.

### 2. Match Row (`MatchRow.client.tsx`)
- **Anatomy (LTR mental model — `<html dir>` handles HE):**
  ```
  ┌──────────────────────────────────────────────────────────────────────┐
  │ [🏴 BRA]   [− 0 +] : [− 0 +]   [🏴 SWI]                   12:00 GMT+3 │
  │                                              ✓ Saved (transient)     │
  └──────────────────────────────────────────────────────────────────────┘
  ```
- **Container:** `bg-[var(--zc-card)] border border-[var(--zc-border)] rounded-2xl pi-4 pbs-3 pbe-3 mbs-3`. Min row height 64px.
- **Flag:** 32×24px PNG/SVG from existing `teams.iso_code` (or `<span aria-hidden>` with country emoji as a fallback). Followed by team name at `text-base font-normal`.
- **Center cluster:** Two steppers separated by a 4px-gap colon `text-2xl text-[var(--zc-muted-foreground)]`.
- **Kickoff time:** `text-sm text-[var(--zc-muted-foreground)] tabular-nums` rendered via `Intl.DateTimeFormat(locale, {hour:'2-digit', minute:'2-digit', timeZoneName:'short'})`. Aligned to the inline-end of the row, top-right visually.
- **Saved indicator:** Absolute-positioned to the row's inline-end-bottom; a small flex `<span class="inline-flex items-center gap-1 text-sm text-[var(--zc-accent)]"><Check16/>{t('saved')}</span>` that mounts on save success and unmounts via CSS `opacity` transition (fade in 150ms, hold 1400ms, fade out 450ms; total 2000ms). Implementation hint for planner: use a `key={lastSavedAt}` + CSS `animation-name` for auto-cleanup without React timers.
- **Error indicator:** Replaces the saved indicator with `text-[var(--zc-destructive)]` text reading `{t('saveFailed')}` (copy below). No timeout — sticks until next successful save.

### 3. Locked Capsule (`MatchRow` locked variant)
- **Anatomy:** `[🏴 BRA] [capsule]  🔒 [🏴 SWI]   12:00 GMT+3`. Center cluster replaced by:
  ```html
  <span dir="ltr" class="bs-8 inline-flex items-center justify-center pi-3 rounded-full bg-[var(--zc-muted)] text-base font-bold text-[var(--zc-primary)] tabular-nums">2 : 1</span>
  <span class="ms-2 text-base text-[var(--zc-muted-foreground)]" aria-label="locked">🔒</span>
  ```
- **No-prediction variant:** Same capsule shape but content `- : -` rendered in `text-[var(--zc-muted-foreground)]`, capsule background `bg-transparent` `border border-[var(--zc-border)]`.
- **Lock emoji:** Unicode `🔒` (U+1F512). Renders consistently on iOS, Android, modern desktop. Do NOT inline an SVG — the emoji is intentionally informal/friendly (family-pool tone) and saves implementation time.
- **Swap timing:** Reload-driven (not animated). The row is re-rendered by `revalidatePath` after Save Result, so the swap is invisible to the player who already submitted; players hitting `/matches` after kickoff get the locked variant on first paint. No transition.

### 4. Post-Result Reveal Row (`MatchRow` resulted variant)
- **Anatomy (vertical, full-bleed under the row):**
  ```
  ┌──────────────────────────────────────────────────────────────────────┐
  │ [🏴 BRA] 2 : 1 [🏴 SWI]   FT · Actual                  12:00 GMT+3   │
  ├──────────────────────────────────────────────────────────────────────┤
  │ User A: 2-1  +4 exact                                                │
  │ User B: 1-1  +2 winner                                               │
  │ User C: 0-2  +0                                                      │
  │ User D: —    +0                                                      │
  └──────────────────────────────────────────────────────────────────────┘
  ```
- **Top sub-row:** Same flag-and-name geometry as locked capsule; center cluster is the ACTUAL score (capsule background `bg-[var(--zc-primary)] text-[var(--zc-primary-foreground)]`). The label `FT · Actual` (EN) / `סיום · תוצאה` (HE) replaces the kickoff time at the inline-end.
- **Bottom block:** One row per family member, sorted by points DESC then name ASC (locale-aware). Each row is `pbs-2 pbe-2 ps-4 pe-4 text-base`, height 32px minimum, separated by `border-t border-[var(--zc-border)]` between players, NOT around the block.
- **Per-player row pattern:** `<span class="font-bold">{name}</span>: <span dir="ltr" class="tabular-nums">{h}-{a}</span>  <PtsBadge points={p} kind={k} />` — the `: ` and double-space are literal characters; the name reads naturally bidi-correct because `<html dir>` controls overall direction.
- **Player with no prediction:** Score replaced by an em dash `—` (U+2014). Badge is `+0` styled `--zc-score-miss`.
- **Reveal mechanism:** Always-expanded. NO accordion in Phase 2 — once a result is entered, the row stays expanded forever. Saves one interaction state, saves one animation timing, saves the "I can't find my score" support hit. If row count becomes overwhelming (>30 resulted matches scrolled), v2 may add per-row collapse.

### 5. Pts Badge (`PtsBadge.tsx`)
- **Anatomy:** `<span class="inline-flex items-center gap-1 ps-2 pe-2 pbs-1 pbe-1 rounded-full text-sm font-bold tabular-nums">+{N} {kind_label}</span>`. Self-contained, no border.
- **Variants:**
  - `kind='exact'`: `text-[var(--zc-score-exact)]` (primary navy) + bold. Label: `exact` / `מדויק`.
  - `kind='goal-diff'`: `text-[var(--zc-score-good)]` + bold. Label: `goal-diff` / `הפרש`.
  - `kind='winner'`: `text-[var(--zc-score-good)]` + bold. Label: `winner` / `מנצח`.
  - `kind='miss'`: `text-[var(--zc-score-miss)]` + normal weight. Label: `(missed)` / `(לא קלע)` — kept in parens to lower visual weight.
- The plus sign is literal (`+`) — render as part of the number with `<span dir="ltr">+{N}</span>` so HE doesn't flip it.

### 6. Sticky Countdown Banner (`CountdownBanner.client.tsx`)
- **Anatomy:** A 40px-tall bar fixed below the 56px header. Z-index 30 (below header z-40). Total sticky chrome = 96px.
- **Container:** `fixed inset-bs-14 inset-i-0 z-30 bs-10 bg-[var(--zc-card)] border-b-1 border-[var(--zc-border)] pi-4 flex items-center justify-between`.
- **Left/start text:** `<span class="text-sm font-bold text-[var(--zc-primary)] truncate">Next: BRA vs SWI</span>` — truncates with ellipsis if team names are long; team names use the active-locale name (`teams.name_en` or `name_he`).
- **Right/end text:** `<span class="text-base font-bold tabular-nums text-[var(--zc-primary)]" dir="ltr">02:14:33</span>` — six-digit `HH:MM:SS` until <1h, then `MM:SS` for the last hour. Always LTR.
- **Tick behavior:** Client component computes from a server-passed `kickoffISO`. `setInterval(1s)`. When `kickoffAt - now <= 0`, the component swaps to the next upcoming fixture (also passed from RSC as a list). When no upcoming fixtures, the banner unmounts (returns `null`) and the page top-padding adjusts.
- **Snap to next fixture:** On rollover, the banner content swaps WITHOUT animation (snap). The page below MUST NOT reflow — the banner stays the same height. The DOM swap is a key change on the inner content.
- **Escalation cue:** When `remaining <= 60s`, the seconds digits color shifts to `text-[var(--zc-accent)]` (per Accent reserved list item 7). No blinking — pure color change. Resets to primary when banner snaps to next fixture.
- **Page offset:** `/matches` page content begins at `mbs-24` (96px = 56 header + 40 banner). When the banner unmounts (post-tournament), the layout snaps to `mbs-14` (56px header only) — the executor must put this conditional spacing on the page wrapper, NOT on every component.

### 7. Date Group Header (`DateGroupHeader.tsx`)
- **Anatomy:** `<h3 class="sticky inset-bs-24 z-20 bs-10 bg-[var(--zc-surface)] pi-4 flex items-center text-sm font-bold text-[var(--zc-muted-foreground)]">{localized date}</h3>`.
- **Sticky offset:** 96px (`inset-bs-24`) — directly below header + countdown banner. When countdown banner hides post-tournament, the header re-sticks at 56px via the page wrapper conditional (planner picks: easiest is two CSS classes via a `data-` attribute on the wrapper).
- **Format:** `Intl.DateTimeFormat(locale, { weekday: 'long', month: 'long', day: 'numeric' })` — yields `Saturday, June 14` (EN) / `יום שבת, 14 ביוני` (HE). Replace the `,` separator with ` · ` post-format (per CONTEXT.md D-01 spec) via simple `.replace(', ', ' · ')`.
- **Background:** Surface (`--zc-surface`) not card — sticky headers SHOULD blend into the page background, not punch out like cards. Adds visual rhythm against the white match cards.

### 8. Leaderboard Row (`LeaderboardRow.client.tsx`)
- **Anatomy (collapsed):**
  ```
  ┌──────────────────────────────────────────────────────────────────────┐
  │ 1   User A                                                  27   ▼  │
  └──────────────────────────────────────────────────────────────────────┘
  ```
- **Container:** Full-width row, `bg-[var(--zc-card)] border-b-1 border-[var(--zc-border)] pi-4 pbs-4 pbe-4 min-bs-14 flex items-center gap-4 cursor-pointer`. First row in the list adds `border-t-1`. Whole row is the tap target — meets 56px min height.
- **Rank:** `<span class="text-base font-bold text-[var(--zc-muted-foreground)] tabular-nums is-6 text-center">{rank}</span>` — 24px-wide fixed column so all ranks left-align. No badge or circle — keep it minimal.
- **Display name:** `<span class="text-base font-normal text-[var(--zc-primary)] flex-1 truncate">{name}</span>`.
- **Total points:** `<span class="text-2xl font-bold text-[var(--zc-primary)] tabular-nums" dir="ltr">{total}</span>` — display-sized to anchor visual hierarchy.
- **Chevron:** Inline SVG `<ChevronDown size=16 class="text-[var(--zc-muted-foreground)]">`. Rotates 180° via `transform rotate-180` when row is expanded. Transition 150ms ease.
- **Expanded sub-block:** Slides into existence via CSS `max-block-size` transition from 0 → auto-equivalent (use the height-transition trick or `display: grid; grid-template-rows: 1fr` toggle). Duration 200ms. Content:
  ```
  League: 18 │ Bracket: 0 — opens June 27 │ Props: 9
  ```
  Rendered as a single flex row at small viewports; wraps to 3 stacked rows below 360px. Padding `pi-4 pbs-3 pbe-3`, background `bg-[var(--zc-muted)]`, text `text-sm font-normal text-[var(--zc-primary)]`. The Bracket "0 — opens June 27" string lives in `messages/{en,he}.json` (copywriting table below).
- **Single-expand rule:** Implemented via a parent-scoped `useState<number | null>(activeRow)`. Tapping a different row closes the current and opens the new one in a single render — no two-row open state ever.

### 9. Flag Grid (`FlagGrid.client.tsx` — for props team picker)
- **Anatomy:** 6 columns × 8 rows = 48 cells. Each cell is a tap target.
- **Container:** `grid grid-cols-6 gap-2 pi-2` (8px gaps, 8px outer padding) so on a 360px-wide viewport each cell computes to ~50px = comfortably above the 44px floor.
- **Cell (unselected):** `aspect-square bs-auto is-full rounded-xl border border-[var(--zc-border)] bg-[var(--zc-card)] flex flex-col items-center justify-center gap-1 ps-1 pe-1 pbs-1 pbe-1`. Contains `<span class="text-2xl leading-none">{flag-emoji-or-img}</span>` + `<span class="text-sm font-normal text-[var(--zc-muted-foreground)] truncate">{teams.name_he|en}</span>` (truncate is critical — country names like "Saudi Arabia" / "ערב הסעודית" overflow at 50px without it).
- **Cell (selected):** Same anatomy, plus: `ring-2 ring-[var(--zc-accent)] ring-offset-2 ring-offset-[var(--zc-card)] bg-[var(--zc-muted)]`. The text color upgrades to `text-[var(--zc-primary)] font-bold`. Cells that are NOT selected dim to `opacity-60` when ANY cell is selected. Selected cell stays at 100%.
- **Tap behavior:** Tap = single-select (radio). Tap selected = deselect. Tap a different cell = swap. Same debounced-save semantics as the stepper (600ms).
- **Locked reveal state (post-first-kickoff):** Grid disappears; replaced by a per-player block matching the post-result reveal pattern (Component 4) — `User A: 🇧🇷 BRA +5 ✓` / `User B: 🇫🇷 FRA +0 (missed)`. Same `PtsBadge` component.

### 10. Saved Inline Checkmark (`SavedIndicator.client.tsx`)
- **Anatomy:** `<span class="inline-flex items-center gap-1 text-sm font-normal text-[var(--zc-accent)]" role="status" aria-live="polite"><CheckSVG16 />{t('saved')}</span>`. The check is an inline SVG checkmark (a 16×16 viewbox, stroke 2, color currentColor). Text: `Saved` / `נשמר`.
- **Lifecycle:** Mount on save success. Auto-unmount after 2000ms via CSS `animation` (no `setTimeout`). Animation: 150ms fade in → 1400ms hold at opacity 1 → 450ms fade out. Re-keys on each successful save so back-to-back saves reset the cycle.

### 11. Admin View/Entry Mode Toggle (`AdminModeToggle.client.tsx`)
- **Anatomy:** Segmented control, 2 segments, full-width-ish on mobile.
  ```
  ┌──────────────────────┬──────────────────────┐
  │   View Mode  (•)     │   Score Entry Mode   │
  └──────────────────────┴──────────────────────┘
  ```
- **Container:** `inline-flex bs-12 rounded-xl bg-[var(--zc-muted)] p-1` (4px inner padding for a "pill on track" effect).
- **Each segment:** `flex-1 bs-10 ps-4 pe-4 inline-flex items-center justify-center rounded-lg text-base font-bold transition-colors duration-150`. Active segment: `bg-[var(--zc-card)] text-[var(--zc-primary)] shadow-sm`. Inactive: `bg-transparent text-[var(--zc-muted-foreground)]`.
- **Active-segment accent:** When `Score Entry Mode` is active, its inner pill gets a 2px-wide accent left/start border (`border-s-2 border-[var(--zc-accent)]`) as a "danger-ish" cue without using destructive red. View Mode active has no accent stripe.
- **Placement:** Top of `/admin/matches` page, below the admin header chrome, above the date-grouped match list. Sticky? No — let it scroll away; the admin doesn't need to flip mode mid-scroll.
- **State persistence:** `URLSearchParam` `?mode=view|entry`. Default = `view`. Survives back/forward and link sharing.

### 12. Admin Score Entry Inputs (in-row, replaces stepper on admin pages in Entry Mode)
- **Anatomy:** Two `<input type="number" inputMode="numeric" min="0" max="9" dir="ltr" class="bs-12 is-12 text-center text-2xl font-bold rounded-xl border border-[var(--zc-border)] focus-visible:border-[var(--zc-accent)] tabular-nums">`, separated by a `:`, followed by `<button class="bs-12 ps-4 pe-4 rounded-xl bg-[var(--zc-primary)] text-[var(--zc-primary-foreground)] font-bold focus-visible:ring-2 focus-visible:ring-[var(--zc-ring)]">Save Result</button>`.
- **Save behavior:** On click, fires the bulk-UPSERT Server Action (D-18). Optimistic UI: button shows "Saving…" disabled state for ≤500ms, then row re-renders as a resulted row via `revalidatePath`.
- **No debounce** — admin save is explicit (intentional confirmation moment). Player save is debounced. This asymmetry is the point.

### 13. Integrity Widget (`IntegrityWidget.tsx`)
- **Anatomy (single line on desktop, wraps to 3 lines on mobile):**
  ```
  Database Sync: OK  │  Total Predictions: 240  │  Unscored Completed Matches: 0
  ```
- **Container:** `fixed inset-be-0 inset-i-0 z-30 bs-10 bg-[var(--zc-primary)] text-[var(--zc-primary-foreground)] pi-4 flex items-center justify-between gap-4 text-sm font-normal tabular-nums` (admin-only — no bottom tab bar in `/admin/*` so this can take the same z-30/40 floor slot).
- **Each metric:** `<span><span class="font-bold">{label}:</span> {value}</span>`.
- **Database Sync OK state:** Value renders `<span class="text-[var(--zc-integrity-ok)] font-bold">OK ✓</span>`.
- **Database Sync FAIL state:** Value renders `<button class="text-[var(--zc-destructive)] font-bold underline">✗ ({N} rows past kickoff)</button>` — the click expands a dropdown above the widget showing offending rows with kickoff timestamps and user IDs. Z-index 31 for the dropdown.
- **Data source:** Server-component-on-pageload (no polling). Three SQL queries from `/admin/(protected)/layout.tsx` so every admin page gets the same widget.

### 14. Inline Save-Failure Error
- **Anatomy:** Replaces the SavedIndicator slot. `<span class="text-sm text-[var(--zc-destructive)]" role="alert">{t('saveFailed')}</span>`. No icon (the destructive red carries enough weight). No toast — stays inline next to the stepper that failed.
- **Recovery:** Next successful save replaces it with the SavedIndicator. No dismiss button — error self-clears on success or page reload.

## Motion Timings

Three-token named scale. Every motion in Phase 2 maps to one of these:

| Token | Duration | Easing | Use |
|-------|----------|--------|-----|
| `motion-fast` | 150ms | `cubic-bezier(0.4, 0, 0.2, 1)` (ease-in-out, Tailwind default) | Stepper button depress feedback; chevron rotate on leaderboard expand; segmented-control segment color transition; SavedIndicator fade-in; PtsBadge color transitions if state changes. |
| `motion-medium` | 200ms | Same easing | Leaderboard row expand/collapse height transition. |
| `motion-slow` | 2000ms total | Custom keyframe: 150ms in / 1400ms hold / 450ms out | SavedIndicator full lifecycle (single CSS animation, not three transitions). |

**No motion-slow `setInterval`s for UI animation.** The only `setInterval` allowed is the countdown ticker (1s tick) and the optional stepper long-press auto-repeat (150ms tick). Both are stateful interactions, not "animations."

**Reduced motion:** Honor `prefers-reduced-motion: reduce` — collapse `motion-medium` row expand to 0ms (instant), keep `motion-fast` color transitions (they read as state, not motion), keep `motion-slow` SavedIndicator at 2000ms total (it's an information channel; can't be instant). Implementation: a single `@media (prefers-reduced-motion: reduce)` block in `globals.css` overriding the `motion-medium` transition durations to `0ms`.

## RTL Stress Points (Phase 2 specific)

Phase 1 verified the chrome flips correctly. Phase 2 introduces new bidi traps. Each is declared:

| Surface | Trap | Fix |
|---------|------|-----|
| Score capsule `2:1` | Hebrew page would flip to `1:2` if rendered as plain text inside an RTL paragraph. | Wrap in `<span dir="ltr">2 : 1</span>`. Always LTR. |
| Countdown timer `02:14:33` | Same. | `<span dir="ltr">02:14:33</span>`. |
| `+pts` badge `+4 exact` | The `+` sign could float to the wrong side. | `<span dir="ltr">+4</span>` for the number; `{kind_label}` follows in the page's `dir`. |
| Flag grid 6×8 layout | CSS Grid is direction-aware; first cell visually appears on inline-start side. On `/he/` this means top-right; on `/en/` top-left. Verify country order is identical (alphabetical by `name_{locale}` or by `iso_code` — planner picks ONE; recommendation: alphabetical by `name_{locale}` so HE-readers see Hebrew alphabetical order). |
| Per-player reveal row `User A: 2-1 +4 exact` | The score and badge could fragment under HE. | Wrap score in `<span dir="ltr">{h}-{a}</span>`; badge already has internal LTR wrapping. |
| Admin pages | EN-only per Phase 1 D-05; even when the admin's `profiles.locale = 'he'`, `/admin/*` MUST render with `<html dir="ltr">`. | The admin layout overrides the `dir` attr regardless of viewer locale. Already-established Phase 1 pattern; reuse. |
| Match row column order | The home team should always be the visually-leading team in BOTH locales (it's a structural concept, not a directional one — "home" is identified by team metadata, not screen side). | Match the existing Phase-1 convention: home is first in DOM order; `<html dir>` swaps which side is "leading." Player intuition adapts to locale. |
| Date header `שבת · 14 ביוני` | The bullet separator and Latin numeric "14" inside Hebrew text. | Native HE bidi handles it; the Hebrew month name "ביוני" is itself an isolation boundary. Use `Intl.DateTimeFormat('he-IL', ...).replace(', ', ' · ')`. Verify in QA-03. |
